### PART-017: Feedback Integration
**Purpose:** Audio feedback and sticker animations on correct/incorrect answers.
**Condition:** Game has audio feedback or sticker moments.
**API (preload):** `await FeedbackManager.sound.preload([{ id, url }, ...])` тАФ NO `register()` method
**API (play):** `await FeedbackManager.sound.play('correct_tap', { subtitle, sticker: <URL string> })` тАФ `sticker` is a plain string URL; FeedbackManager wraps it into the package's internal `{ image, duration, type: 'IMAGE_GIF' }` shape. Game code MUST pass a string. Object form is a stale/internal shape.
**Key rules:**
- **Sequential audio (CRITICAL).** Any function body awaiting 2+ FeedbackManager audio calls wraps in `FeedbackManager.runSequence(async () => { ... })`. CTA / screen-change handlers keep calling `sound.stopAll()` + `stream.stopAll()` тАФ those now also abort the ambient `runSequence`. Legacy `audioStopped` flag is forbidden. Validator: `GEN-FEEDBACK-RUN-SEQUENCE`. See `feedback/reference/feedbackmanager-api.md` ┬з Sequential Audio.
- Simple games: preload `correct_tap` + `wrong_tap`
- Multi-part games: preload `all_correct`, `partial_correct_attempt1`, `partial_correct_last_attempt`, `all_incorrect_attempt1`, `all_incorrect_last_attempt`
- Sticker: pass a **string URL** to the `sticker` parameter; FeedbackManager wraps it internally. Do NOT construct the `{ image, duration }` object yourself тАФ that is the package-internal shape.
- Use `preload()` with array of `{id, url}` тАФ there is NO `register()` method
- **Audio failure handling is package-internal (CRITICAL).** The FeedbackManager package bounds every failure path at 2.5s and resolves with a status object тАФ playing audio runs to its natural end with no max-duration cap. Game code calls `FeedbackManager.sound.play(...)` and `FeedbackManager.playDynamicFeedback(...)` DIRECTLY inside `try/catch`. No wrappers (`safePlaySound`, `safeTTS`, `audioRace`), no `Promise.race`, no minimum-duration floors. Validator `5e0-FEEDBACK-RACE-FORBIDDEN` rejects any `Promise.race` wrapping a FeedbackManager call. Failure status shapes: `{status:'ok'}`, `{status:'timeout', reason:'api'|'stream-setup'|'first-chunk'|'jit-load'}`, `{status:'error', reason:'api-status'|'stream-setup'|'stream'|'unexpected', ...}`, `{status:'missing', id}`, `{status:'locked', id}`, `{status:'aborted'}`. The `try/catch` exists only to absorb `AbortError` from `runSequence` cancellation. **Submit-handler TTS MUST be awaited (CRITICAL).** Inside answer/submit/round-complete handlers, both `FeedbackManager.sound.play(...)` AND `FeedbackManager.playDynamicFeedback(...)` are awaited inside `try/catch`. Validator: `GEN-FEEDBACK-TTS-AWAIT`. See PART-026 Anti-Pattern 36. **Carve-outs that may stay fire-and-forget:** round-start TTS in `showRoundIntro` / `onMounted` (student should interact immediately), chain-progress / partial-match audio, tile-select / ambient SFX. Do NOT re-enable inputs in the submit handler; `renderRound()` / `loadRound()` is the single source of truth for re-enabling.
- **No leftover feedback between rounds / at end of game (CRITICAL).** No audio, subtitle, or sticker from the previous round may be audible or visible when the next round renders or the end screen appears. Every `nextRound()`, `scheduleNextRound()` timeout body, `endGame()` (FIRST line after the `gameEnded` guard), `restartGame()`, and level-transition `action` callback MUST call `try { FeedbackManager.sound.stopAll(); } catch(e) {} try { FeedbackManager.stream.stopAll(); } catch(e) {}` BEFORE mutating `gameState` for the new phase. The FeedbackManager overlay auto-clear fires only when a new `playDynamicFeedback()` starts тАФ silent auto-advance and end-screen entry do NOT trigger it, so previous-round feedback bleeds through without an explicit stop. If the game renders subtitle/sticker in custom DOM outside the FeedbackManager overlay, also reset `textContent = ''` and remove `show`/`correct`/`incorrect`/`visible` classes in the same block. Do NOT `.remove()` cached DOM references (GEN-DOM-CACHE). Validator rule `5e0-CLEANUP-BETWEEN-ROUNDS` enforces this at build time. See Feedback SKILL Student-Visible Invariants and `skills/feedback/reference/timing-and-blocking.md` ┬з "Round/Phase Cleanup".
